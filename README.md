# Neura - Telegram AI Bot

A production-ready Telegram bot with AI responses, conversation history, and webhook deployment on Railway.

## Features

- 🤖 AI-powered responses via OpenAI-compatible API (OpenRouter, OpenAI, local LLMs)
- 💾 Conversation history per user/group stored in Supabase
- ⚡ FastAPI webhook server for reliable Telegram updates
- 🔍 Inline query support
- 🚀 Deploy-ready for Railway with automatic webhook registration

## Prerequisites

- Python 3.11+
- Telegram Bot Token (from [@BotFather](https://t.me/BotFather))
- Supabase project (for conversation history)
- LLM API credentials (OpenRouter, OpenAI, or any OpenAI-compatible endpoint)

## Quick Start

### 1. Clone & Install

```bash
git clone <your-repo>
cd neura-bot
cp .env.example .env
# Edit .env with your credentials
pip install -r requirements.txt
```

### 2. Configure Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `TELEGRAM_BOT_TOKEN` | Yes | Bot token from @BotFather |
| `WEBHOOK_URL` | Yes* | Public HTTPS URL (e.g., `https://your-app.railway.app`) |
| `PORT` | No | Port to bind (Railway sets automatically, default 8000) |
| `SUPABASE_URL` | Yes | Supabase project URL |
| `SUPABASE_SERVICE_ROLE_KEY` | Yes | Supabase service role key |
| `LLM_API_URL` | Yes | OpenAI-compatible chat completions endpoint |
| `LLM_API_KEY` | Yes | API key for LLM provider |
| `LLM_MODEL` | No | Model name (default: `openai/gpt-4o-mini`) |
| `BOT_NAME` | No | Bot display name (default: `Neura`) |
| `BOT_SYSTEM_PROMPT` | No | System prompt for AI personality |

*Required for production webhook. For local dev with ngrok, set to your ngrok HTTPS URL.

### 3. Supabase Setup

Create a `conversations` table in your Supabase database:

```sql
create table conversations (
  id bigint generated by default as identity primary key,
  user_id bigint not null,
  group_id bigint not null,
  role text not null check (role in ('user', 'assistant')),
  message text not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

create index conversations_user_group_created_idx
  on conversations (user_id, group_id, created_at asc);
```

Enable Row Level Security if desired (service role key bypasses RLS).

### 4. Local Development

For local development, you need a public HTTPS URL for Telegram webhooks. Use ngrok:

```bash
# Terminal 1: Start ngrok
ngrok http 8000

# Terminal 2: Set WEBHOOK_URL to your ngrok HTTPS URL in .env, then run
uvicorn main:app --reload
```

### 5. Deploy to Railway

1. Push this repo to GitHub
2. Connect repo in Railway dashboard
3. Add all environment variables in Railway Variables tab
4. Deploy — Railway automatically provides `$PORT` and HTTPS domain
5. Webhook is registered automatically on startup

### Verify Webhook

```bash
curl "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getWebhookInfo"
```

Should show your webhook URL with `pending_update_count: 0`.

## Architecture

- **Webhook mode**: No polling — Telegram POSTs updates to `/webhook`
- **Stateless**: FastAPI handles requests; python-telegram-bot processes updates
- **History**: Per `(user_id, group_id)` composite key in Supabase
- **Inline queries**: Use `group_id=0` sentinel (isolated from chat history)

## Commands

- `/start` — Welcome message
- Any text message — AI reply with history context
- Inline query `@neura_bot <query>` — AI reply as inline result
